# 故障排查指南

> **说明**：本文档涵盖 PySide6 桌面应用可能遇到的常见问题及解决方案。部分问题来源于 PySide6 / Qt 框架的特性，已标注 `[通用]` 的内容需要根据实际环境调整。

---

## 1. 启动失败类

### 1.1 应用无法启动

**错误现象**：双击应用图标或运行 `python app/main.py` 后无反应

**可能原因**：

| 原因 | 检查步骤 | 解决方案 |
|------|---------|---------|
| Python 版本错误 | `python --version` | 确保 Python >= 3.10 |
| 虚拟环境未激活 | `which python` 检查路径 | 激活虚拟环境 |
| 依赖未安装 | `pip list | grep PySide6` | 运行 `pip install -r requirements.txt` |
| PySide6 DLL 缺失 | 查看错误信息 | 安装 Visual C++ Redistributable |
| 多 Python 版本冲突 | `python --version` 和 `python3 --version` | 使用正确的 Python 解释器 |

**排查命令**：

```powershell
# Windows
python --version
pip list | Select-String PySide6

# macOS/Linux
python3 --version
pip3 list | grep PySide6
```

### 1.2 数据库初始化失败

**错误现象**：启动时提示"数据库初始化失败"

**可能原因**：

| 原因 | 检查步骤 | 解决方案 |
|------|---------|---------|
| 数据库文件被锁定 | 检查是否多进程同时访问 | 关闭其他实例 |
| 写入权限不足 | 检查数据目录权限 | 更改数据目录路径或权限 |
| 数据库文件损坏 | 检查错误信息中的 SQLite 错误 | 删除数据库文件后重新初始化 |
| 迁移未执行 | `alembic current` 检查 | 运行 `alembic upgrade head` |

**排查命令**：

```bash
# 检查数据库文件
ls -la ~/.novel-writer/novel_writer.db

# 检查数据库完整性
sqlite3 ~/.novel-writer/novel_writer.db "PRAGMA integrity_check;"

# 删除后重新初始化
rm ~/.novel-writer/novel_writer.db  # Linux/macOS
del %USERPROFILE%\.novel-writer\novel_writer.db  # Windows
python -m app.cli db init
```

### 1.3 窗口显示异常

**错误现象**：应用启动但界面显示异常（空白、布局错乱、白屏）

**可能原因**：

| 原因 | 检查步骤 | 解决方案 |
|------|---------|---------|
| QSS 文件缺失 | 检查 `ui/styles/dark.qss` 是否存在 | 确认样式文件完整 |
| 屏幕 DPI 设置 | 检查系统缩放设置 | 设置 `QT_AUTO_SCREEN_SCALE_FACTOR=1` |
| 显卡驱动问题 | 检查 Qt 渲染引擎警告 | 设置 `QT_QUICK_BACKEND=software` |
| 窗口布局缓存损坏 | 检查 `layout.ini` 缓存 | `python app/main.py --reset-layout` |
| 字体缺失 | 检查控制台字体相关错误 | 安装缺少的中文字体 |

**排查命令**：

```bash
# 设置环境变量后重新启动（解决 DPI/渲染问题）
QT_AUTO_SCREEN_SCALE_FACTOR=1 QT_QUICK_BACKEND=software python app/main.py

# Windows 设置环境变量
$env:QT_AUTO_SCREEN_SCALE_FACTOR="1"
$env:QT_QUICK_BACKEND="software"
python app/main.py
```

### 1.4 缺少 Qt 平台插件

**错误现象**：`This application failed to start because no Qt platform plugin could be initialized.`

**可能原因**：

| 原因 | 检查步骤 | 解决方案 |
|------|---------|---------|
| PySide6 损坏 | `pip check pyside6` | 重新安装 PySide6 |
| 环境变量缺失 | 检查 `QT_QPA_PLATFORM_PLUGIN_PATH` | 设置正确的插件路径 |
| 打包后路径错误 | 检查 PyInstaller 打包配置 | 正确包含 platform 插件 |

**解决方案**：

```bash
# 重新安装 PySide6
pip uninstall PySide6
pip install PySide6

# 手动设置平台插件路径
$env:QT_QPA_PLATFORM_PLUGIN_PATH="venv/Lib/site-packages/PySide6/plugins/platforms"

# 验证安装
python -c "from PySide6.QtWidgets import QApplication; print('OK')"
```

## 2. AI 调用类

### 2.1 API Key 无效或过期

**错误现象**：AI 操作时提示认证错误

**可能原因**：

| 原因 | 检查步骤 | 解决方案 |
|------|---------|---------|
| API Key 未设置 | 检查 `.env` 文件 | 添加对应的 API Key |
| API Key 格式错误 | 检查 Key 的前缀和长度 | 重新复制 API Key |
| API Key 已过期 | 在提供商平台检查 | 重新生成 API Key |
| 选择了错误的提供商 | 检查应用的 AI 提供商设置 | 切换到正确的提供商 |

**代码中的错误处理**：

```python
# services/ai_service.py
class AIService(QObject):
    ai_error = Signal(str, str)  # (error_type, message)
    
    async def safe_generate(self, prompt, **kwargs):
        try:
            return await self.ai_manager.generate(prompt, **kwargs)
        except AuthError:
            self.ai_error.emit("auth", "API Key 无效或已过期，请在设置中检查")
        except Exception as e:
            self.ai_error.emit("unknown", f"AI 调用失败: {str(e)}")
```

**排查命令**：

```bash
# 检查环境变量是否设置
python -c "from dotenv import load_dotenv; load_dotenv(); import os; print('OpenAI Key:', os.getenv('OPENAI_API_KEY', '未设置')[:10] + '...')"

# 测试 AI 连接
python -m app.cli ai test --provider openai
# 预期输出：连接成功 或 认证失败（请检查 API Key）
```

### 2.2 请求超时

**错误现象**：AI 请求长时间无响应或超时

**可能原因**：

| 原因 | 检查步骤 | 解决方案 |
|------|---------|---------|
| 网络连接问题 | `ping api.openai.com` | 检查网络或切换网络 |
| 需要代理 | 检查系统代理设置 | 设置 `HTTP_PROXY` / `HTTPS_PROXY` |
| API 服务器负载高 | 查看状态页面 | 稍后重试 |
| 超时设置过短 | 检查 AI 配置 | 增加超时时间 |
| 模型不可用 | 查看提供商状态 | 切换到其他模型 |

**配置超时参数**：

```python
# core/ai/providers/base.py
@dataclass
class AIConfig:
    timeout: int = 60  # 默认 60 秒
    # ...

# 在设置中可调整
# 建议值：普通 30-60 秒，大模型 60-120 秒
```

**排查命令**：

```bash
# 测试网络连通性
ping api.openai.com

# 测试 API 响应时间
curl -w "耗时: %{time_total}s\n" -o /dev/null -s https://api.openai.com/v1/models

# 检查代理设置
echo $env:HTTP_PROXY
echo $env:HTTPS_PROXY
```

### 2.3 配额不足（429 错误）

**错误现象**：收到 429 错误或"请求过于频繁"

**可能原因**：

| 原因 | 检查步骤 | 解决方案 |
|------|---------|---------|
| API 调用次数超限 | 检查提供商使用配额 | 等待配额重置或升级套餐 |
| 速率限制 | 检查 `Retry-After` 头 | 自动等待后重试 |
| 免费账户限制 | 查看账户等级 | 升级付费账户 |

**自动重试机制**：

```python
# core/ai/error_handler.py
@with_retry(max_retries=3, base_delay=2.0)
async def generate_with_retry(self, messages):
    return await self.provider.chat(messages)

# 重试策略：
# 第 1 次失败 → 等待 2 秒
# 第 2 次失败 → 等待 4 秒
# 第 3 次失败 → 等待 8 秒
# 超过 3 次 → 提示用户
```

### 2.4 生成内容异常

**错误现象**：AI 返回内容不完整、格式错误、或不符合要求

**可能原因**：

| 原因 | 检查步骤 | 解决方案 |
|------|---------|---------|
| 提示词过长 | 检查 Token 计数 | 减少上下文或使用更长上下文的模型 |
| 温度过高 | 检查当前温度设置 | 降低温度到 0.5-0.7 |
| 模型能力不足 | 检查输出质量 | 切换到更强大的模型 |
| 提示词冲突 | 检查系统提示词 | 减少矛盾指令 |
| 敏感内容过滤 | 检查内容是否触及底线 | 调整写作内容 |

**排查方法**：

```python
# 1. 检查 Token 用量
token_count = token_counter.count_messages(messages)
print(f"提示词共 {token_count} tokens")
# 建议：gpt-4o 最大 128k，claude 最大 200k

# 2. 测试不同温度
# 续写建议：temperature=0.8
# 润色建议：temperature=0.6
# 分析建议：temperature=0.3

# 3. 使用 Prompt 验证器
audit_result = audit_service.audit_text(ai_response)
if audit_result["total_issues"] > 0:
    print("AI 生成内容存在以下问题：")
    for issue in audit_result["issues"]:
        print(f"  - [{issue['severity']}] {issue['suggestion']}")
```

### 2.5 Ollama 本地模型问题

**错误现象**：Ollama 连接失败或模型加载慢

**可能原因**：

| 原因 | 检查步骤 | 解决方案 |
|------|---------|---------|
| Ollama 服务未启动 | `ollama list` | 运行 `ollama serve` |
| 模型未下载 | `ollama list` 检查已下载模型 | `ollama pull qwen2.5:14b` |
| 端口冲突 | 检查 11434 端口是否被占用 | 修改 Ollama 端口配置 |
| 内存不足 | 检查系统资源使用 | 使用更小的模型（如 7B 代替 14B） |
| GPU 未启用 | 检查 Ollama 日志 | 配置 GPU 加速 |

**排查命令**：

```bash
# 检查 Ollama 服务状态
ollama list

# 测试 Ollama API
curl http://localhost:11434/api/tags

# 拉取推荐模型

ollama run qwen2.5:14b "你好"
```

## 3. 数据库类

### 3.1 SQLite 文件锁定

**错误现象**：`database is locked` 错误

**可能原因**：

| 原因 | 检查步骤 | 解决方案 |
|------|---------|---------|
| 多进程同时访问 | 检查是否有其他实例运行 | 关闭其他实例 |
| 长时间事务 | 检查是否有事务未提交 | 确保短事务、及时提交 |
| WAL 模式未启用 | 检查 PRAGMA 设置 | 启用 WAL 模式 |

**解决方案**：

```python
# models/database.py - 确保 WAL 模式已启用
def init_db(self, db_path: str):
    self.engine = create_engine(
        f"sqlite:///{db_path}",
        connect_args={"check_same_thread": False}
    )
    with self.engine.connect() as conn:
        conn.execute(text("PRAGMA journal_mode=WAL"))
        conn.execute(text("PRAGMA busy_timeout=5000"))  # 5 秒超时
```

### 3.2 查询性能慢

**错误现象**：打开项目或搜索时响应慢

**可能原因**：

| 原因 | 检查步骤 | 解决方案 |
|------|---------|---------|
| 缺少索引 | 检查 EXPLAIN QUERY PLAN | 添加合适的索引 |
| 章节内容过大 | 检查数据库大小 | 大文本分离存储 |
| 数据库碎片 | VACUUM 后测试 | 定期 VACUUM |

**排查命令**：

```bash
# 检查数据库大小
python -c "
import os
size = os.path.getsize(os.path.expanduser('~/.novel-writer/novel_writer.db'))
print(f'数据库大小: {size/1024/1024:.1f} MB')
if size > 50 * 1024 * 1024:
    print('建议：数据库超过 50MB，考虑优化')
"

# 手动 VACUUM
python -c "
from sqlalchemy import create_engine, text
engine = create_engine('sqlite:///' + os.path.expanduser('~/.novel-writer/novel_writer.db'))
with engine.connect() as conn:
    conn.execute(text('VACUUM'))
    print('VACUUM 完成')
"
```

## 4. UI 显示类

### 4.1 编辑器中文乱码

**错误现象**：编辑器中中文显示为乱码

**可能原因**：

| 原因 | 检查步骤 | 解决方案 |
|------|---------|---------|
| 字体缺失 | 检查系统是否安装中文字体 | 安装思源宋体/微软雅黑 |
| 编码问题 | 检查文件编码 | 确保使用 UTF-8 保存 |
| PySide6 字符集 | 检查 Qt 区域设置 | 设置正确的 locale |

**解决方案**：

```python
# ui/editor/text_editor.py
from PySide6.QtCore import QLocale

# 在应用入口设置
app = QApplication(sys.argv)
QLocale.setDefault(QLocale(QLocale.Chinese, QLocale.China))

# 编辑器字体设置
font = QFont("Source Han Serif SC", 16)
font.setStyleStrategy(QFont.PreferAntialias)  # 抗锯齿
editor.setFont(font)
```

### 4.2 界面卡顿

**错误现象**：操作界面时响应慢、动画卡顿

**可能原因**：

| 原因 | 检查步骤 | 解决方案 |
|------|---------|---------|
| 主线程阻塞 | 检查是否有耗时操作在主线程 | AI 调用/大文件保存使用异步 |
| 动画过多 | 检查 QSS 动画 | 减少动画效果 |
| 内存泄漏 | 监控内存使用 | 检查对象引用和信号连接 |
| 大型章节文件 | 检查章节大小 | 按需加载，分页显示 |

**排查工具**：

```bash
# 使用 Python 性能分析器
python -m cProfile -o profile_output.prof app/main.py

# 使用 Qt 性能分析
$env:QT_LOGGING_RULES="*.debug=true; qt.qpa.*=false"
python app/main.py
```

## 7. 常见问题速查表

| 错误信息 | 原因 | 解决方案 |
|---------|------|---------|
| `ModuleNotFoundError: No module named 'PySide6'` | PySide6 未安装 | `pip install PySide6` |
| `database is locked` | 数据库被锁定 | 关闭其他实例，或等待后重试 |
| `API key not found` | API Key 未设置 | 在设置中配置 API Key |
| `Connection refused: Ollama` | Ollama 服务未启动 | 运行 `ollama serve` |
| `Qt platform plugin not initialized` | Qt 平台插件缺失 | 重新安装 PySide6，或设置平台路径 |
| `FileNotFoundError: dark.qss` | 主题文件缺失 | 确认 `ui/styles/` 目录完整 |
| `ImportError: DLL load failed` | VC++ 运行时缺失 | 安装 Visual C++ Redistributable |
| `Alembic: Target database is not up to date` | 数据库需要迁移 | 运行 `alembic upgrade head` |
| `Token limit exceeded` | 上下文过长 | 减少上下文内容或使用更长上下文的模型 |
| `429 Too Many Requests` | API 调用频率超限 | 等待后自动重试 |
