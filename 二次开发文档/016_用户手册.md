# 用户手册 — Novel Writer 使用指南

> **适用版本**：PySide6 版 v1.0.0  
> **面向读者**：小说创作者（无需技术背景）

---

## 一、快速上手

### 1.1 安装应用

#### Windows

```powershell
# 1. 从官网下载 NovelWriter-Setup.exe
# 2. 双击运行安装程序
# 3. 选择安装目录（建议默认）
# 4. 完成安装，桌面出现"Novel Writer"图标
```

#### macOS

```bash
# 1. 下载 NovelWriter.dmg
# 2. 双击打开 DMG 文件
# 3. 将 NovelWriter 拖入 Applications 文件夹
# 4. 首次打开需在"安全性与隐私"中允许运行
```

### 1.2 初次启动

1. 双击桌面图标启动应用
2. 首次启动会自动创建数据目录（`~/.novel-writer/`）
3. 进入主界面后，点击 **文件 → 新建项目** 开始创作

## 二、项目管理

### 2.1 新建项目

**操作路径**：文件 → 新建项目 / `Ctrl+N`

**填写内容**：

| 字段 | 说明 | 建议 |
|------|------|------|
| **项目名称** | 你的小说名称 | 如《仙途》、《星海征途》 |
| **项目路径** | 存储位置 | 建议放在专门的小说文件夹 |
| **小说类型** | 作品分类 | 玄幻、仙侠、都市、科幻等 |
| **写作方法** | 创作方法论 | 新手选"三幕式"，进阶选"英雄之旅" |
| **预计字数** | 目标总字数 | 网文通常 100-300 万字 |
| **项目简介** | 小说简介 | 一句话概括故事 |

> 写作方法可以后续随时切换，不影响已写内容。

### 2.2 打开项目

**操作路径**：文件 → 打开项目 / `Ctrl+O`

快捷方式：
- **最近打开**：文件 → 最近打开 → 选择项目
- **直接拖拽**：将项目文件夹拖入主窗口

### 2.3 项目设置

**操作路径**：项目 → 项目设置

可调整的项目级配置：
- 项目名称和简介
- 小说类型
- 写作方法
- 封面图片
- 导出设置

## 三、写作基础

### 3.1 创建章节

**方式一**：在左侧项目树中，右键点击 → 新建章节  
**方式二**：使用菜单 → 项目 → 章节管理 → 新建

**分卷管理**：章节数超过 20 章后，建议使用分卷归类：
- 右键分卷列表 → 新建分卷
- 拖拽章节到不同分卷

### 3.2 编辑器使用

编辑器支持 **富文本格式** 和 **纯文本** 两种模式。

**工具栏按钮**：

| 按钮 | 快捷键 | 功能 |
|:----:|:------:|------|
| **B** | `Ctrl+B` | 加粗 |
| *I* | `Ctrl+I` | 斜体 |
| **H1** | `Ctrl+Shift+1` | 一级标题 |
| **H2** | `Ctrl+Shift+2` | 二级标题 |
| 🔍 | `Ctrl+F` | 搜索 |
| 🔄 | `Ctrl+H` | 替换 |

**编辑器设置**（视图 → 字体大小）：
- `Ctrl++`：放大字体
- `Ctrl+-`：缩小字体
- `Ctrl+0`：重置字体

### 3.3 自动保存

编辑器默认每 **30 秒** 自动保存当前内容。状态栏会显示保存状态：
- ○ = 内容已更改，等待保存
- ● = 已保存

你也可以随时按 `Ctrl+S` 手动保存。

### 3.4 字数统计

状态栏实时显示：
- **总字数**：项目所有章节的总字数
- **当前章节字数**：编辑中章节的字数
- **段落数**：当前章节的段落统计

## 四、AI 辅助写作

### 4.1 配置 AI

首次使用 AI 功能前，需要进行配置：

**操作路径**：AI → AI 设置

**步骤**：
1. 选择 AI 提供商（如 OpenAI、Anthropic Claude）
2. 输入 API Key（点击输入框右侧的眼睛图标可查看明文）
3. 选择模型（推荐 GPT-4o 或 Claude Sonnet）
4. 点击"测试连接"验证
5. 点击确定保存

> API Key 会加密存储在本地，不会上传到任何第三方。

**支持的提供商**：

| 提供商 | 开源/免费 | 推荐模型 | 使用成本 |
|--------|:---------:|---------|:--------:|
| OpenAI | ❌ | GPT-4o | 按 Token 计费 |
| Anthropic | ❌ | Claude Sonnet 4 | 按 Token 计费 |
| Google | ❌ | Gemini 2.0 Flash | 有免费额度 |
| DeepSeek | ⚠️ | DeepSeek Chat | 价格较低 |
| **Ollama（本地）** | **✅** | **Qwen2.5:14b** | **完全免费** |

> 推荐预算有限的用户使用 **Ollama + 本地模型**，完全免费且数据不出本机。

### 4.2 AI 续写

**操作方式**：
- 在编辑器中将光标定位到需要续写的位置
- 按 `Ctrl+Enter` 或点击工具栏"AI 续写"按钮
- AI 生成的文本会逐字"打字"插入到编辑器中

**控制**：
- `Esc`：中断生成
- `Enter`：确认保存生成的内容
- 生成完成后继续编辑，内容自动保留

### 4.3 AI 润色

**操作方式**：
1. 在编辑器中选择需要润色的文字
2. 按 `Ctrl+Shift+P` 或右键 → AI 润色
3. 在弹出的窗口中选择润色风格（更生动/更简洁/更抒情等）
4. 查看 AI 润色结果（原稿 vs 修改稿对比）
5. 点击"接受"或"拒绝"

### 4.4 AI 对话

**操作路径**：AI → AI 对话面板 / `Ctrl+Shift+A`

在底部 AI 对话窗口中，你可以：
- 向 AI 提问创作问题
- 讨论情节发展
- 请求灵感建议
- 获取写作指导

### 4.5 七步创作法

AI 支持一整套专业创作流程（原版核心功能继承）：

| 步骤 | 命令 | 功能 | 操作路径 |
|:----:|:----:|------|---------|
| 1 | 创作宪法 | 定义创作核心原则 | 写作 → 创作宪法 |
| 2 | 故事规格 | 定义故事核心要素 | 写作 → 故事规格 |
| 3 | 决策澄清 | 澄清关键决策点 | 写作 → 决策澄清 |
| 4 | 创作计划 | 制定创作技术方案 | 写作 → 创作计划 |
| 5 | 任务分解 | 分解为可执行任务 | 写作 → 任务分解 |
| 6 | AI 续写 | 使用以上设定进行写作 | `Ctrl+Enter` |
| 7 | 质量分析 | AI 分析内容质量 | 写作 → 质量分析 |

> 推荐新手按 1→7 的顺序执行一次完整流程，建立创作框架。

## 五、角色管理

### 5.1 添加角色

**操作路径**：右侧栏 → 角色面板 → 添加角色

**必填信息**：
- 角色姓名
- 角色定位（主角、配角、反派等）

**可选信息**：性别、年龄、外貌、性格、背景、技能、秘密等

### 5.2 编辑角色

在角色列表中双击角色名称，可查看和编辑角色的完整档案。

### 5.3 角色状态追踪

系统会自动记录角色在各章节的：
- **出场记录**：角色在哪些章节出现过
- **状态变化**：存活状态、心理状态、位置变化
- **成长阶段**：起点 → 发展 → 转折 → 高潮 → 完成

### 5.4 角色关系

在关系图谱中查看角色之间的关系网络：
- 关系类型：盟友、敌人、恋情、家族等
- 关系强度：1-10 级
- 关系变化历史：关系如何随时间演变

## 六、情节管理

### 6.1 情节节点

根据你选择的写作方法，系统会生成对应的情节结构。

例如"三幕式"的结构：
```
第一幕 (开端)
  ├── 开场
  ├── 激励事件
  └── 第一幕转折点
第二幕 (中段)
  ├── 次要情节
  ├── 中点转折
  └── 第二幕转折点
第三幕 (结尾)
  ├── 高潮前奏
  ├── 高潮
  └── 结局
```

你可以为每个节点关联到具体章节，追踪创作进度。

### 6.2 伏笔管理

在追踪面板记录和管理伏笔：
- **埋设**：记录在哪个章节埋下伏笔
- **揭示**：记录在哪个章节揭晓伏笔
- **状态**：待揭示 / 已揭示 / 已废弃

### 6.3 冲突管理

记录故事中的冲突：
- 冲突类型：人与人的冲突、人与社会的冲突等
- 升级程度：1-10 级，随剧情发展逐步升级
- 解决方式：如何解决冲突

## 七、世界观构建

### 7.1 添加世界观设定

**操作路径**：右侧栏 → 追踪面板 → 世界观标签

支持的设定类型：
- **地点**：国家、城市、建筑、自然景观
- **规则**：魔法体系、科技设定、社会规则
- **物品**：神器、武器、重要道具
- **事件**：历史事件、传说故事

### 7.2 设定关联

每个世界观设定可以关联到：
- 特定角色（谁与此设定相关）
- 其他设定（父子关系、同级关联）
- 标签分类（方便搜索）

---

## 八、时间线管理

### 8.1 添加时间线事件

**操作路径**：右侧栏 → 追踪面板 → 时间线标签

记录故事内的时间事件：
- 事件名称
- 故事内时间（如"第 3 年 5 月"）
- 对应章节
- 参与者
- 重要性

### 8.2 时间线验证

系统会自动检查：
- 时间冲突：同一时间发生矛盾的剧情
- 时序错误：事件顺序不合逻辑
- 角色年龄：角色年龄是否与时间线一致

---

## 九、一致性检查

### 9.1 角色一致性

系统会在每次写作后自动检查：
- **姓名一致性**：角色名称前后一致
- **称呼一致性**：角色间的称呼不变化
- **行为一致性**：角色行为符合性格设定

### 9.2 关系一致性

- 关系变化是否被记录
- 关系状态是否合理
- 关系发展轨迹是否清晰

### 9.3 世界观规则一致性

- 设定的规则是否被遵守
- 是否有违反世界观的剧情

## 十、导出与分享

### 10.1 导出项目

**操作路径**：文件 → 导出 → 选择格式

**支持的导出格式**：

| 格式 | 用途 | 导出速度（10 万字） |
|:----:|------|:------------------:|
| **纯文本 (.txt)** | 通用备份、分享 | < 2 秒 |
| **Markdown (.md)** | Git 版本管理、发布到创作平台 | < 2 秒 |
| **HTML (.html)** | 网页直接预览 | < 3 秒 |
| **EPUB (.epub)** | 导入 Kindle、iBooks 等阅读器 | < 10 秒 |
| **PDF (.pdf)** | 打印、正式提交、存档 | < 15 秒 |
| **Word (.docx)** | 交给编辑、进一步排版 | < 8 秒 |

### 10.2 从原版导入

如果你之前使用过原版 Novel Writer（CLI 版），可以无缝迁移：

**操作路径**：文件 → 从原版导入 → 选择原版项目目录

系统会自动导入：
- 章节内容
- 角色档案
- 情节追踪数据
- 关系网络
- 时间线
- 世界观设定

| 主题 | 适用场景 | 特点 |

## 十三、常见问题

### 13.1 AI 连不上

**现象**：AI 续写一直转圈无结果

**排查步骤**：
1. 检查网络连接是否正常
2. 检查 API Key 是否正确（AI → AI 设置 → 测试连接）
3. 检查 API 提供商是否有服务中断公告
4. 检查是否达到了 API 调用配额限制

### 13.2 编辑器卡顿

**现象**：打字延迟、滚动不流畅

**解决方案**：
1. 关闭右侧不使用的面板（视图 → 切换右侧栏）
2. 降低编辑器字体大小
3. 将超大章节拆分为多个小章节

### 13.3 应用无法启动

**现象**：双击图标后无反应

**排查步骤**：
1. 以管理员身份运行（右键 → 以管理员身份运行）
2. 查看日志文件：`%USERPROFILE%\.novel-writer\logs\`
3. 重装应用
4. 如果仍然无法解决，提交 Issue 并附上日志文件

### 13.4 如何反馈问题

**提供以下信息有助于快速定位**：
1. 操作系统版本
2. 应用版本（帮助 → 关于）
3. 完整的错误截图
4. 操作步骤（做了什么导致了问题）
