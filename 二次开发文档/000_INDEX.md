## 🏗️ 整体架构规划

### 技术栈

| 层级 | 技术选型 | 说明 |
|------|---------|------|
| **UI 层** | PySide6 (Qt Widgets) | 原生桌面界面 |
| **业务逻辑层** | Python 3.10+ | 核心引擎 |
| **数据层** | SQLite + SQLAlchemy | 本地数据库 |
| **AI 层** | LangChain + LiteLLM | 统一 AI 接口 |
| **样式** | QSS (Qt StyleSheet) | 界面美化 |
| **打包** | PyInstaller | 生成可执行文件 |

## 📝 转换原则

1. **保留核心逻辑**：写作方法论、提示词模板、追踪系统等核心逻辑全部保留
2. **适配新技术栈**：UI 层、数据层、插件系统需要重新设计
3. **增强 AI 能力**：利用 Python AI 生态，增加更多 AI 功能
4. **优化用户体验**：原生桌面应用的优势要充分发挥
5. **保持数据兼容**：文件格式尽量与原版兼容，支持导入
