## 9. 技术债务与维护策略

### 9.1 技术债务跟踪

| 债务项 | 影响 | 计划解决版本 |
|--------|------|------------|
| 富文本编辑器复杂度 | 编辑器功能迭代慢 | v1.1 |
| 大章节性能 | 超长章节编辑卡顿 | v1.0-rc |
| 插件 API 稳定性 | 插件兼容性 | v1.0-rc |
| 异步任务管理 | 后台任务监控不完善 | v1.1 |
| 测试覆盖率不足 | 核心模块需要覆盖 | v1.0-rc |

### 9.2 依赖更新策略

| 更新类型 | 频率 | 检查方式 | 操作 |
|---------|:----:|---------|------|
| **安全漏洞修复** | 即时 | `safety check -r requirements.txt` | 发现后 24 小时内更新 |
| **补丁版本** | 每月 | `pip list --outdated` | 自动合并（CI 验证） |
| **次版本更新** | 每季度 | 人工审查 CHANGELOG | 评估后手动更新 |
| **主版本更新** | 每年 | 全面兼容性测试 | 规划迁移窗口 |
| **PySide6 更新** | 每 Qt 6 新版本 | Qt 官方发布通知 | 1 个月内适配 |
| **Python 版本** | 每 Python 新大版本 | Python 官方发布 | 6 个月内适配 |

**具体执行**：

```bash
# 月度依赖检查脚本
#!/bin/bash
# scripts/check-deps.sh
echo "=== 依赖安全检查 ==="
safety check -r requirements.txt

echo "=== 过期依赖列表 ==="
pip list --outdated

echo "=== 许可证变化 ==="
pip-licenses --format=markdown --no-dev | diff - licenses-report.md

echo "=== 建议操作 ==="
echo "• 补丁更新 → 直接更新"
```

**CI 自动化依赖检查**：

```yaml
# .github/workflows/deps.yml
name: Dependency Check
on:
  schedule:
    - cron: "0 6 1 * *"  # 每月 1 日
  workflow_dispatch:

jobs:
  check:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-python@v5
        with: { python-version: "3.11" }
      - run: pip install safety pip-licenses
      - run: safety check -r requirements.txt --continue-on-error
      - run: pip-licenses --format=markdown --no-dev > licenses-current.md
      - run: |
          if ! diff licenses-report.md licenses-current.md > /dev/null 2>&1; then
            echo "许可证有变化，请审查！"
          fi
```
