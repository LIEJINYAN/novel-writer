# 数据备份与恢复

> **说明**：PySide6 版采用 SQLite 数据库存储主要数据，同时保留部分文件存储（章节大文本、导出的文件等）。以下备份策略需要根据实际安装路径调整。

---

## 1. 需要备份的内容

### 1.1 应用级数据（全局）

| 路径/文件 | 说明 | 优先级 | 备份必要性 |
|-----------|------|--------|-----------|
| `~/.novel-writer/novel_writer.db` | 应用级数据库（项目列表、AI 提供商配置等） | 🔴 高 | **必须备份** |
| `~/.novel-writer/config.json` | 全局配置（主题、语言、窗口布局等） | 🟡 中 | 建议备份 |
| `~/.novel-writer/plugins/` | 已安装的插件配置 | 🟡 中 | 建议备份 |
| `~/.novel-writer/logs/` | 日志文件 | 🟢 低 | 无需备份 |

### 1.2 项目级数据

每个项目独立存储，默认位置在项目目录下的 `.novel/` 目录：

| 路径/文件 | 说明 | 优先级 | 备份必要性 |
|-----------|------|--------|-----------|
| `.novel/project.db` | 项目数据库（章节内容、角色、追踪数据等） | 🔴 高 | **必须备份** |
| `.novel/config.json` | 项目配置 | 🔴 高 | **必须备份** |
| `.novel/memory/` | 创作记忆（宪法、风格参考） | 🟡 中 | 建议备份 |
| `stories/` | 章节内容（文件系统存储） | 🔴 高 | **必须备份** |
| `exports/` | 导出文件 | 🟢 低 | 无需备份 |

### 1.3 环境配置文件

| 文件 | 说明 | 备份必要性 |
|------|------|-----------|
| `.env` | API Key 等环境变量 | **必须备份**（注意安全存储） |
| `venv/` | Python 虚拟环境 | 无需备份（可重新创建） |

### 1.4 备份内容总结

```
🔴 高优先级（必须备份）：
  ~/.novel-writer/novel_writer.db
  每个项目下的 .novel/project.db
  每个项目下的 .novel/config.json
  每个项目下的 stories/（章节内容）
  .env（环境变量）

🟡 中优先级（建议备份）：
  ~/.novel-writer/config.json
  ~/.novel-writer/plugins/
  每个项目下的 .novel/memory/

🟢 低优先级（可选备份）：
  ~/.novel-writer/logs/
  exports/
  venv/
  node_modules/
```

## 3. 备份命令

### 3.1 使用内置备份功能（推荐）

```bash
# 备份所有项目和应用数据
python -m app.cli backup create --all

# 备份指定项目
python -m app.cli backup create --project /path/to/project

# 备份到指定目录
python -m app.cli backup create --all --output D:\backups\novel-writer

# 查看备份列表
python -m app.cli backup list

# 备份配置：排除大文件（如图片资源）
python -m app.cli backup create --all --exclude "*.png,*.jpg,*.mp4"
```

### 3.2 手动备份 - Windows PowerShell

```powershell
# 创建备份目录
$backupDir = "D:\backups\novel-writer-$(Get-Date -Format 'yyyyMMdd-HHmmss')"
New-Item -ItemType Directory -Force -Path $backupDir

# 备份应用级数据库
Copy-Item "$env:USERPROFILE\.novel-writer\novel_writer.db" "$backupDir\app_data\"
Copy-Item "$env:USERPROFILE\.novel-writer\config.json" "$backupDir\app_data\"

# 备份项目数据
$projectDir = "C:\Users\liejin\Documents\my-novel"
Copy-Item -Recurse "$projectDir\.novel" "$backupDir\projects\my-novel\"
Copy-Item -Recurse "$projectDir\stories" "$backupDir\projects\my-novel\"

# 备份环境变量
Copy-Item "$projectDir\.env" "$backupDir\"

# 压缩备份
Compress-Archive -Path "$backupDir\*" -DestinationPath "$backupDir.zip"
Remove-Item -Recurse $backupDir

Write-Host "备份完成: $backupDir.zip"
```

### 3.3 手动备份 - macOS/Linux

```bash
#!/bin/bash
# backup-novel-writer.sh

BACKUP_DIR="$HOME/backups/novel-writer"
TIMESTAMP=$(date +%Y%m%d-%H%M%S)
BACKUP_FILE="$BACKUP_DIR/novel-writer-backup-$TIMESTAMP.tar.gz"

mkdir -p "$BACKUP_DIR"

# 应用数据
APP_DATA_DIR="$HOME/.novel-writer"

# 项目目录列表（请根据实际路径修改）
PROJECTS=(
    "$HOME/Documents/my-novel"
)

tar -czvf "$BACKUP_FILE" \
    "$APP_DATA_DIR/novel_writer.db" \
    "$APP_DATA_DIR/config.json" \
    $(for p in "${PROJECTS[@]}"; do
        echo "$p/.novel"
        echo "$p/stories"
        echo "$p/.env"
    done)

echo "备份完成: $BACKUP_FILE"
echo "大小: $(du -h "$BACKUP_FILE" | cut -f1)"
```

### 3.4 SQLite 直接备份（数据库级）

```bash
# 使用 SQLite 的 .backup 命令（推荐，不影响正在使用的数据库）

# 备份应用数据库
sqlite3 ~/.novel-writer/novel_writer.db ".backup 'backup_app.db'"

# 备份项目数据库
sqlite3 /path/to/project/.novel/project.db ".backup 'backup_project.db'"

# 使用 Python 脚本备份
python -c "
import sqlite3
import shutil
from datetime import datetime

timestamp = datetime.now().strftime('%Y%m%d_%H%M%S')

# 在线备份（不影响正在运行的数据库）
src = sqlite3.connect('path/to/database.db')
dst = sqlite3.connect(f'backup_{timestamp}.db')
src.backup(dst)
dst.close()
src.close()
print(f'数据库备份完成: backup_{timestamp}.db')
"
```

## 4. 备份频率建议

| 使用情况 | 应用数据 | 项目数据 | 说明 |
|---------|---------|---------|------|
| **日常写作** | 每周 | 每次完成后 | 使用内置备份命令 |
| **重大修改** | 操作前 | 操作前 | 如修改项目结构、升级版本 |
| **定期** | 每月 | 每周 | 使用定时任务 |
| **长期不写作** | 无需 | 每月 | 避免过时备份 |

### 4.1 定时任务设置

#### Windows 任务计划程序

```powershell
# 创建备份脚本 backup_novel.ps1
$backupDir = "$env:USERPROFILE\backups\auto"
$timestamp = Get-Date -Format 'yyyyMMdd-HHmmss'
New-Item -ItemType Directory -Force -Path $backupDir

# 执行备份
python -m app.cli backup create --all --output "$backupDir\$timestamp"

# 保留最近 30 个备份，删除旧备份
Get-ChildItem $backupDir -Filter "*.zip" | 
    Sort-Object LastWriteTime -Descending | 
    Select-Object -Skip 30 | 
    Remove-Item -Force
```

然后在"任务计划程序"中创建每周任务执行此脚本。

#### Linux/macOS crontab

```bash
# 编辑 crontab
crontab -e

# 每周日凌晨 3 点自动备份
0 3 * * 0 /home/user/scripts/backup-novel-writer.sh

# 每月 1 号凌晨 3 点执行完整备份
0 3 1 * * /home/user/scripts/backup-novel-full.sh
```

#### 使用内置定时备份

```bash
# 启用定时自动备份（默认每天）
python -m app.cli backup schedule --enable --interval daily

# 设置保留备份数量
python -m app.cli backup schedule --retain 30

# 查看备份计划状态
python -m app.cli backup schedule --status

# 禁用定时备份
python -m app.cli backup schedule --disable
```

## 5. 恢复步骤

### 5.1 从内置备份恢复

```bash
# 查看可用备份
python -m app.cli backup list

# 恢复所有数据
python -m app.cli backup restore --latest

# 恢复特定时间的备份
python -m app.cli backup restore --timestamp 20260701_120000

# 恢复单个项目
python -m app.cli backup restore --project my-novel --timestamp 20260701_120000

# 恢复前先预览（不实际恢复）
python -m app.cli backup restore --latest --dry-run
```

### 5.2 手动恢复 - Windows

```powershell
# 1. 解压备份文件
# 使用 7-Zip 或 Expand-Archive
Expand-Archive -Path "D:\backups\novel-writer-20260701-120000.zip" -DestinationPath "D:\backups\temp"

# 2. 关闭应用（确保数据库未在使用）

# 3. 恢复应用数据库
Copy-Item "D:\backups\temp\app_data\novel_writer.db" "$env:USERPROFILE\.novel-writer\"

# 4. 恢复项目数据库
Copy-Item "D:\backups\temp\projects\my-novel\.novel\project.db" "C:\Users\liejin\Documents\my-novel\.novel\"
Copy-Item "D:\backups\temp\projects\my-novel\stories" "C:\Users\liejin\Documents\my-novel\" -Recurse

# 5. 恢复环境变量
Copy-Item "D:\backups\temp\.env" "C:\Users\liejin\Documents\my-novel\"

# 6. 清理临时文件
Remove-Item -Recurse "D:\backups\temp"
```

### 5.3 手动恢复 - macOS/Linux

```bash
#!/bin/bash
# restore-novel-writer.sh

BACKUP_FILE="$HOME/backups/novel-writer-backup-20260701-120000.tar.gz"
RESTORE_DIR="/tmp/novel-restore"

# 1. 解压
mkdir -p "$RESTORE_DIR"
tar -xzvf "$BACKUP_FILE" -C "$RESTORE_DIR"

# 2. 停止应用

# 3. 恢复应用数据
cp "$RESTORE_DIR/home/user/.novel-writer/novel_writer.db" "$HOME/.novel-writer/"
cp "$RESTORE_DIR/home/user/.novel-writer/config.json" "$HOME/.novel-writer/"

# 4. 恢复项目数据
cp "$RESTORE_DIR/home/user/Documents/my-novel/.novel/project.db" "$HOME/Documents/my-novel/.novel/"
cp -r "$RESTORE_DIR/home/user/Documents/my-novel/stories/"* "$HOME/Documents/my-novel/stories/"

# 5. 恢复环境变量
cp "$RESTORE_DIR/home/user/Documents/my-novel/.env" "$HOME/Documents/my-novel/.env"

# 6. 验证恢复
echo "验证数据库完整性..."
sqlite3 "$HOME/.novel-writer/novel_writer.db" "PRAGMA integrity_check;"
sqlite3 "$HOME/Documents/my-novel/.novel/project.db" "PRAGMA integrity_check;"

# 7. 清理
rm -rf "$RESTORE_DIR"
echo "恢复完成！"
```

### 5.4 从原版项目导入

```bash
# 如果原版项目数据（JSON/MD 格式）需要导入到 PySide6 版本
python -m app.cli import --source /path/to/original/project

# 单独导入章节内容
python -m app.cli import chapters --source /path/to/original/stories

# 导入追踪数据
python -m app.cli import tracking --source /path/to/original/spec/tracking
```

## 6. 云同步方案

### 6.1 使用网盘同步

> **注意**：仅同步备份文件，不要直接同步运行中的数据库文件。

| 云存储 | 操作说明 |
|--------|---------|
| **OneDrive** | 将备份目录设置为 OneDrive 同步文件夹 |
| **Google Drive** | 使用 Google Drive 桌面版同步备份目录 |
| **iCloud** | 将备份目录加入 iCloud 云盘 |
| **阿里云盘** | 使用阿里云盘桌面版同步 |
| **百度网盘** | 使用百度网盘桌面版同步 |

### 6.2 使用 rclone（跨平台）

```bash
# 安装 rclone
# Windows: choco install rclone
# macOS: brew install rclone
# Linux: sudo apt install rclone

# 配置云端存储
rclone config

# 同步备份目录到云端
rclone sync -P \
  ~/backups/novel-writer \
  remote:novel-writer-backups

# 从云端恢复
rclone sync -P \
  remote:novel-writer-backups \
  ~/backups/novel-writer

# 设置定时同步（配合 crontab）
# 每天凌晨 4 点同步
0 4 * * * rclone sync -P ~/backups/novel-writer remote:novel-writer-backups
```

### 6.3 使用 Git 版本控制

```bash
# 创建独立的 Git 仓库用于管理项目数据
cd /path/to/project

# 初始化 Git（如果尚未初始化）
git init

# 创建 .gitignore（只备份数据文件，排除临时文件）
cat > .gitignore << 'EOF'
.novel/project.db-wal
.novel/project.db-shm
__pycache__/
*.pyc
venv/
node_modules/
exports/
.DS_Store
Thumbs.db
EOF

# 添加数据文件
git add .novel/project.db
git add .novel/config.json
git add .novel/memory/
git add stories/
git add .env.example

# 提交
git commit -m "备份: 2026-07-02 写作数据"

# 推送到远程仓库
git remote add origin https://github.com/username/novel-data.git
git push -u origin main
```

> **重要**：不要将包含 API Key 的 `.env` 文件提交到 Git。使用 `.env.example` 作为模板。

## 8. 备份策略建议

### 8.1 推荐方案

```
频率          类型            保留数量         说明
─────────────────────────────────────────────────────
每次写作       增量备份         30 次            自动保存的历史版本
每日          自动备份         7 天             内置定时备份
每周          全量备份         4 周             脚本自动创建
每月          完整归档         12 个月          手动 + 云存储

总保留策略:
- 最近 7 天：每日备份
- 最近 4 周：每周备份
- 最近 12 个月：每月备份
- 年度归档：永久保存
```

### 8.2 存储建议

| 存储方式 | 适用备份类型 | 说明 |
|---------|-------------|------|
| **本地磁盘** | 日常、增量 | 速度快，容量大 |
| **外置硬盘** | 每周、月度 | 物理隔离，安全 |
| **云存储** | 月度、归档 | 异地容灾 |
| **Git 仓库** | 项目数据 | 版本历史 |

### 8.3 注意事项

1. **不要在写作时直接备份**：应在应用关闭后再执行备份
2. **加密敏感信息**：API Key、密码等应加密存储
3. **测试恢复流程**：建议每月测试一次备份恢复
4. **多地备份**：至少保留一份本地备份和一份云端备份
5. **记录备份日志**：记录备份时间、内容、大小

## 9. 紧急恢复检查表

当数据丢失或损坏时，按以下步骤操作：

| 步骤 | 操作 | 说明 |
|------|------|------|
| 1 | **停止写入** | 立即关闭应用，避免覆盖数据 |
| 2 | **保存当前状态** | 复制当前数据目录 |
| 3 | **检查可用备份** | `python -m app.cli backup list` |
| 4 | **选择备份版本** | 根据时间选择最近的备份 |
| 5 | **预览恢复** | `--dry-run` 确认不会覆盖需要的数据 |
| 6 | **执行恢复** | 运行恢复命令 |
| 7 | **验证完整性** | 确认数据库无损坏 |
| 8 | **启动应用** | 验证所有功能正常 |
| 9 | **立即备份** | 恢复后立即创建新备份 |
| 10 | **记录事件** | 记录故障原因和恢复过程 |

---

> ⚠️ **重要提醒**：备份文件只有在成功恢复时才有意义。**请定期手动测试备份恢复流程**，确保备份文件的可用性。建议在新建项目后立即创建一次完整备份，验证备份和恢复流程正常工作。
