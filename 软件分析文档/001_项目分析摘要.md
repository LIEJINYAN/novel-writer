### 4.4 实时 AI API 集成（stardust-dreams 插件）

**这是项目中唯一直接调用 AI API 的模块**，其他模块均通过 AI 助手平台的命令模板间接使用 AI。

#### 4.4.1 API 客户端

| 文件 | 路径 | 职责 |
|------|------|------|
| API客户端 | `plugins/stardust-dreams/lib/api-client.js` | 与星尘织梦服务端通信，获取加密 Prompt 和会话数据 |
| 解密器 | `plugins/stardust-dreams/lib/decryptor.js` | 在内存中解密加密的 Prompt，确保不持久化 |
| 安全存储 | `plugins/stardust-dreams/lib/secure-storage.js` | 加密存储认证信息，基于设备特征生成密钥 |
| Prompt管理器 | `plugins/stardust-dreams/lib/prompt-manager.js` | 协调获取、解密、填充 Prompt 的完整流程 |
| 模板引擎 | `plugins/stardust-dreams/lib/template-engine.js` | 变量替换、条件渲染、循环等模板处理 |

#### 4.4.2 API 端点

| 端点 | 方法 | 用途 |
|------|------|------|
| `/api/trpc/form.getSession` | POST | 获取会话信息 |
| `/api/trpc/form.getPrompt` | POST | 获取加密的 Prompt（核心功能） |

#### 4.4.3 安全机制

1. **加密传输**：Prompt 通过 AES-256-GCM 加密后传输
2. **内存解密**：解密结果仅在内存中，永不持久化到磁盘
3. **设备绑定**：认证信息使用设备特征（主机名、用户名等）生成唯一密钥加密存储
4. **定期清理**：内存中的敏感数据使用后立即清理，设置定时器定期检查内存使用

## 6. API路由与命令定义

> **说明**：本项目是**CLI工具**，**无HTTP API路由**。所有交互通过以下三种方式进行：
> 1. **CLI命令**：终端中使用 `novel` 命令执行项目初始化、插件管理等操作
> 2. **AI斜杠命令**：在AI助手（如Claude、Cursor、Gemini）内部使用 `/xxx` 命令进行小说创作
> 3. **实时API调用**：仅 stardust-dreams 插件通过 HTTP API 调用星尘织梦服务

### 6.1 CLI命令

| 命令 | 子命令 | 用途 |
|------|--------|------|
| `novel init` | - | 初始化新小说项目 |
| `novel check` | - | 检查系统环境和AI工具 |
| `novel plugins` | - | 插件管理总览 |
| `novel plugins:list` | - | 列出已安装插件 |
| `novel plugins:add` | `<name>` | 安装插件 |
| `novel plugins:remove` | `<name>` | 移除插件 |
| `novel upgrade` | - | 升级现有项目 |
| `novel info` | - | 查看可用写作方法 |
| `novel help` | `<command>` | 显示帮助信息 |

### 6.2 AI斜杠命令（在AI助手内部使用）

**核心七步方法论：**
- `/constitution` - 创建创作宪法，定义核心原则
- `/specify` - 定义故事规格，明确要创造什么
- `/clarify` - 澄清关键决策点，明确模糊之处
- `/plan` - 制定技术方案，决定如何创作
- `/tasks` - 分解执行任务，生成可执行清单
- `/write` - AI辅助写作章节内容
- `/analyze` - 综合验证分析，确保质量一致

**追踪管理：**
- `/plot-check` - 检查情节一致性
- `/timeline` - 管理故事时间线
- `/relations` - 追踪角色关系变化
- `/track` - 综合追踪与智能分析
- `/track-init` - 初始化追踪系统

**专家模式：**
- `/expert` - 列出可用专家
- `/expert plot` - 剧情结构专家
- `/expert character` - 人物塑造专家

## 7. 项目架构总结

### 7.1 架构特点

1. **CLI优先设计**：核心功能通过命令行工具提供，适合开发者和技术型写作者
2. **多AI平台适配**：支持13个AI助手平台，通过命令模板自动生成各平台配置
3. **插件化扩展**：8个内置插件，支持功能扩展和风格定制
4. **方法论驱动**：6种经典写作方法，支持智能推荐和方法转换
5. **追踪系统**：完善的情节、角色、关系、时间线追踪机制
6. **反AI检测**：内置反AI检测规范和写作指南

### 7.2 工作流程

```
用户启动 → CLI命令 → 项目初始化 → AI助手打开项目 → 斜杠命令创作 → 脚本执行 → 追踪更新 → 质量审计
```

### 7.3 关键设计模式

| 模式 | 应用 |
|------|------|
| 模板方法 | 写作方法预设（presets） |
| 策略模式 | 方法推荐评分算法 |
| 插件模式 | 功能扩展系统 |
| 工厂模式 | 多AI平台命令生成 |

## 8. 技术栈总结表

| 维度 | 技术 | 说明 |
|------|------|------|
| 语言 | TypeScript | 类型安全，现代语法 |
| 框架 | Commander.js | CLI框架 |
| 交互 | Inquirer.js | 交互式选择 |
| 存储 | JSON/YAML | 文件存储，无数据库 |
| AI集成 | 多平台命令模板 | 通过AI助手内部命令调用 |
| 构建 | TypeScript Compiler | tsc编译 |
| 开发 | tsx | 热重载运行 |

**注意**：本项目**不使用**数据库（无ORM、SQL迁移），所有数据以JSON/YAML文件形式存储。AI调用通过各AI助手平台的命令模板实现，不直接调用AI API。
