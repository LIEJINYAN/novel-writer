# AI 面板双模式重设计（Chat / Agent）

## Why

当前 AI 面板把"自由聊天"和"操作执行"混在一起：用户在聊天框里说"帮我润色第三章"，AI 只会回复文字建议，不会真正去执行润色操作。引入 Agent 模式后，用户可以用自然语言直接指挥 AI 干活——不再需要手动点按钮。

## 新增需求

### FR-1: 工具注册中心
ToolRegistry 管理所有可用工具，每个工具定义：
- `name` — 工具名称（如 `continue_write`）
- `description` — 描述（用于 LLM 理解何时调用）
- `parameters` — 参数 schema（JSON Schema）
- `handler` — 执行函数
- `category` — 分类（read/edit/system/network）

### FR-2: Agent 执行器
AgentExecutor 负责：
1. 接收用户自然语言输入
2. 调用 LLM 判断意图并选择工具
3. 执行工具，获取结果
4. 将结果反馈给 LLM 生成最终回复
5. 结果写入编辑器或弹出对话框
6. 在 UI 中展示工具调用过程（工具名、参数、状态、耗时）

### FR-3: 工具集

编辑类：`continue_write`, `polish`, `rewrite`
阅读类：`analyze_chapter`, `read_chapter`, `detect_ai`
搜索类：`search_web`
预览类：`preview_html`
系统类：`apply_diff`, `show_dialog`

### FR-4: AI 面板双模式切换
- Chat 模式：纯文字对话，不调用工具
- Agent 模式：对话 + 工具调用可视化
- 工具调用过程在消息区展示为可折叠的"工具卡片"
- 模式切换记忆（不重置对话历史）
- 模型选择、AI 设置两个模式共享

## 接受标准

### AC-1: Chat 模式
- 纯文字对话，AI 只回复不操作
- 与当前的对话面板行为一致

### AC-2: Agent 模式
- "帮我润色这段文字" → 自动调用 polish 工具，结果写入编辑器
- "分析第三章" → 自动调用 analyze 工具，结果展示在对话框
- 工具调用过程在消息区可视化展示

### AC-3: 模式切换
- 切换时对话历史保留
- 模型选择、温度设置两个模式共享
- 切换模式不中断正在进行的生成
