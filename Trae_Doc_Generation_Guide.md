### 2. 核心模块逻辑与数据流（ARCHITECTURE.md）

```
请生成 ARCHITECTURE.md 文档，重点说明：
- 核心功能模块列表（如：小说项目管理、编辑器、AI 调用引擎、提示词管理、历史版本等）
- 各模块的输入/输出、核心函数/类
- 模块之间的调用关系（最好用文字描述一个完整的用户操作流程，例如：“用户点击续写 → 前端发送请求 → 后端组装提示词 → 调用 AI API → 返回结果 → 前端渲染”）
- 数据流转示意图（可以使用 Mermaid 序列图或流程图，如果项目适合的话）

### 3. 数据库表结构说明（DATABASE\_SCHEMA.md）

```

  - 字段清单（字段名、类型、是否主键/外键、默认值、约束、说明）

```

### 4. API 接口文档（API\_REFERENCE.md）

```

  - HTTP 方法及路径
  - 功能描述

- 建议使用 Markdown 表格展示，请求/响应使用 JSON 代码块

### 5. AI 调用与提示词模板说明（AI\_PROMPTS.md）

## ✅ 最后建议

- **执行顺序**：先执行第一步（项目分析），然后按顺序生成核心文档，最后根据需要生成补充文档。
- **多轮修正**：如果 Trae 生成的某份文档不符合预期，可以直接追问“请更新 ARCHITECTURE.md 中的数据流描述，增加对 XXX 模块的说明”。
- **保存方式**：建议将生成的内容逐个复制到项目根目录或 `docs/` 文件夹下，并用 Git 提交。后续代码有更新时，可以让 Trae 根据 git diff 增量更新文档。

## 💡 使用提示

- **无需每次都从头执行**：以后需要生成或更新某份文档时，只需打开本文件，复制对应的单条指令即可。
- **可版本管理**：把这本文件也提交到 Git 里，方便追踪你对指令集的优化。
- **可拆分**：如果你觉得指令太长了，也可以拆成多个文件（如 `step1_analysis.md`, `step2_core_docs.md` 等）。
