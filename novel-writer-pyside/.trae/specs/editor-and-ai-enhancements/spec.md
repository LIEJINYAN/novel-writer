# 编辑器增强 & AI 功能完善 - 产品需求文档

## 概述
- **摘要**：基于开发路线图，将 Phase 2（编辑器系统）和 Phase 3（AI 集成）中尚未完成的用户体验优化任务补齐，包括编辑器行号、撤销/重做历史、搜索历史、续写字数选择、API Key 加密升级、自动保存配置、拖拽排序保护和大纲跳转等 8 个功能点。
- **目的**：补齐编辑器核心体验缺失的功能，提升 AI 写作的可控性和安全性
- **目标用户**：网文作者、使用本软件进行小说创作的所有用户

## 目标
- 编辑器增加行号显示，提升定位效率
- 增加撤销/重做历史记录管理（自定义栈深度、Ctrl+Z/Y）
- 自动保存间隔可在设置面板配置
- 续写时可选择字数（500/1000/1500/2000/自定义）
- API Key 加密从异或升级为标准 cryptography Fernet 加密
- 搜索增加历史记录（保留最近 20 条）
- 拖拽排序增加确认提示和撤销功能
- 大纲面板节点点击跳转到对应章节

## 非目标（本次不做）
- 编辑器富文本格式（粗体/斜体/标题等）
- AI 对话历史搜索（远期想法）
- 搜索的全文索引优化（Phase 5 性能优化）
- 大纲面板的写作方法推荐（Phase 3.6 未完成项，但不在此范围）
- 编辑器分屏/对比模式

## 背景与上下文
项目已完成了：Ollama 适配器（httpx）、搜索面板（Ctrl+F）、自动保存定时器（30 秒固定）和项目树拖拽排序（直接生效无确认）。这些功能已有基础但缺少关键使用体验。

## 功能需求

### FR-1: 编辑器行号显示
- 编辑器左侧显示行号，与代码编辑器的行号区域类似
- 行号区域宽度随数字位数自动调整
- 当前行高亮
- 支持暗色/亮色主题

### FR-2: 撤销/重做历史记录管理
- Ctrl+Z / Ctrl+Y 撤销和重做
- 保留历史记录栈（可配置深度）
- 在设置面板中显示历史记录栈深度设置

### FR-3: 自动保存间隔可在设置面板配置
- 默认 30 秒
- 在 AI 设置对话框或通用设置中添加自动保存间隔输入
- 支持 10~300 秒范围

### FR-4: 续写时可选择字数
- 默认 2000 字
- 提供快速选项：500 / 1000 / 1500 / 2000 / 自定义
- 在 AI 面板的续写操作区域增加字数选择控件

### FR-5: API Key 加密用 cryptography 库
- 将 `utils/crypto.py` 的自定义异或加密替换为 `cryptography.fernet.Fernet`
- 添加 `cryptography` 到依赖
- 向后兼容：检测旧格式加密自动重新加密保存

### FR-6: 搜索历史记录
- 保存最近 20 条搜索关键词
- 在搜索输入框聚焦或输入时展示历史下拉
- Ctrl+F 打开搜索面板时恢复上次搜索词
- 支持按上下方向键选择历史项

### FR-7: 拖拽排序确认提示与撤销功能
- 拖拽章节后弹出一个非阻塞提示条（如："章节顺序已调整"）
- 提示条包含"撤销"按钮
- 点击撤销可恢复原排序
- 提示条 5 秒后自动消失

### FR-8: 大纲面板点击跳转
- 大纲节点的章节范围显示为可点击区域
- 点击展开章节列表，点击具体章节跳转到编辑器
- 支持 Ctrl+Click 在新标签页打开

## 非功能需求
- NFR-1：撤销栈深度默认 100 步，最大 500 步
- NFR-2：自动保存间隔更改即时生效，无需重启应用
- NFR-3：搜索历史仅在当前会话中有效，不清除（可选持久化到数据库）
- NFR-4：cryptography 加密的 Key 绑定机器码 + 应用密钥

## 约束
- **技术**：Python 3.11+，PySide6，SQLAlchemy 2.0+
- **依赖**：需添加 `cryptography` 包到 requirements.txt
- **现有代码不得破坏**：所有修改必须向后兼容

## 假设
- 用户希望行号像代码编辑器一样显示在左侧
- 用户对续写字数有明确偏好（500/1000/1500/2000）
- 搜索历史不需要跨会话持久化

## 验收标准

### AC-1: 编辑器行号显示
- **Given** 打开任意章节编辑器
- **When** 查看编辑器左侧
- **Then** 显示连续的行号，行号区域背景与原编辑区域有区分
- **Verification**: `human-judgment`

### AC-2: 撤销/重做
- **Given** 编辑器中有内容
- **When** 按 Ctrl+Z
- **Then** 撤销上一次操作；按 Ctrl+Y 重做
- **Verification**: `programmatic`

### AC-3: 自动保存配置
- **Given** 打开设置面板
- **When** 修改自动保存间隔为 60 秒
- **Then** 自动保存定时器更新为 60 秒间隔
- **Verification**: `programmatic`

### AC-4: 续写字数选择
- **Given** 打开 AI 面板的续写功能
- **When** 在字数选择下拉中选择 500 字
- **Then** AI 续写时按 500 字生成
- **Verification**: `human-judgment`

### AC-5: API Key 加密升级
- **Given** 数据库中存储了加密的 API Key
- **When** 启动应用
- **Then** 使用 Fernet 解密，旧格式自动迁移
- **Verification**: `programmatic`

### AC-6: 搜索历史记录
- **Given** 在搜索面板中搜索过关键词
- **When** 再次点击搜索框
- **Then** 显示搜索历史下拉列表
- **Verification**: `human-judgment`

### AC-7: 拖拽排序确认提示
- **Given** 在项目树中拖拽移动章节
- **When** 放下章节
- **Then** 显示非阻塞提示条，含"撤销"按钮
- **Verification**: `human-judgment`

### AC-8: 大纲跳转章节
- **Given** 大纲面板显示写作方法节点
- **When** 点击节点的章节范围
- **Then** 编辑器跳转到对应章节
- **Verification**: `human-judgment`

## 未解决问题（已评估）
- [x] **搜索历史是否需要跨会话持久化？** → 当前不需要。搜索历史使用 QCompleter 内存实现，每次启动清空。持久化需求低，用户习惯本次会话内的关键词循环。
- [x] **撤销栈深度是否需要用户可配置？** → 是。已在设置对话框中添加 QSpinBox（10~500 步，默认 100），配置通过 QSettings 持久化。
- [x] **大纲面板节点的章节范围如果跨多章，点击是跳转第一章还是展开列表？** → 展开 QMenu 下拉列表，用户选择具体章节后跳转。
